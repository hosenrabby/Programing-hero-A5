Question-3: What is event delegation in the DOM?

Event delegation is a technique in JavaScript where you delegate the responsibility of handling events to a parent element.
How Event Delegation Works
1 - Event Bubbling: When an event occurs on a child element, it "bubbles up" through its parent elements in the DOM tree.
2 - Parent Listener: Instead of attaching event listeners to each child element, you attach a single event listener to a parent element.
3 - Event Target: Inside the event handler, you use the event.target property to identify the specific child element that triggered the event.
4 - Performance: Attaching a single event listener to a parent element is more efficient than attaching multiple listeners to individual child elements, especially for large lists or dynamically added elements.
5 - Simplified Code: Reduces the amount of code and makes it easier to manage.

Question-4: What is event bubbling in the DOM?

Event Bubbling where an event triggered on a child element propagates ("bubbles up") through its ancestor elements in the DOM tree. This means that if an event occurs on a deeply nested element, it will also trigger event listeners on all of its parent elements
1 - Event Target: The event starts at the target element (the element where the event occurred).
2 - Bubbling Phase: The event then propagates upward through the DOM tree, triggering event listeners on each ancestor element.

Question-5: How do you create, add, and remove elements using JavaScript?

1 - Create an Element Use the document.createElement() method to create a new element.
2 - Add Content or Attributes to the Element You can add content, attributes, or styles to the newly created element.
3 - Add the Element to the DOM Use methods like appendChild(), append() and prepend() to add the element to the DOM.
